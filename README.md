# Actividad de GitHub Classroom: Propuesta de Práctica Temática Pequeña

## Descripción general
En esta actividad vas a diseñar una **propuesta de práctica temática pequeña** orientada a programación de sistemas. El foco principal es la documentación técnica, la planeación del alcance y la justificación de diseño del repositorio.

## Objetivo de aprendizaje
Al terminar la actividad, el estudiante será capaz de:
- Definir una necesidad real y convertirla en una propuesta técnica viable.
- Justificar la elección de lenguaje según restricciones de tiempo y complejidad.
- Diseñar una estructura limpia de repositorio para GitHub Classroom.
- Documentar entradas, salidas, pruebas y criterios de éxito antes de programar.

## Lenguajes permitidos
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
1. Resolver un solo problema concreto.
2. Evitar interfaces gráficas, frameworks y arquitecturas complejas.
3. Limitar funciones al mínimo viable (MVP académico).
4. Priorizar scripts, consola y archivos de texto simples.
5. ARM64 Assembly se recomienda solo para programas muy pequeños.
6. No integrar nube, bases de datos ni APIs externas.

## Ejemplos de posibles temas
- **Mini Toolkit en ARM64**
- **Asistente de Estudio en Terminal**
- **Reporteador de Información del Sistema**
- **Organizador de Archivos**
- **Juego de Aprendizaje en Línea de Comandos**

## Entregables esperados
1. Propuesta completa en `docs/propuesta.md`.
2. Caso de uso detallado en `docs/caso_de_uso.md`.
3. Justificación de estructura en `docs/estructura_repositorio.md`.
4. Plan de pruebas en `docs/plan_de_pruebas.md` y checklist en `tests/test_plan.md`.
5. Prototipo mínimo opcional en `src/main.<ext>`.
6. Script de ejecución adaptado en `scripts/run.sh`.

## Instrucciones para el estudiante
1. Elige un tema pequeño y realista.
2. Selecciona lenguaje principal y justifícalo.
3. Completa todos los archivos de `docs/`.
4. Define pruebas manuales suficientes para validar tu idea.
5. (Opcional) Implementa un prototipo mínimo en `src/`.
6. Ajusta `scripts/run.sh` para tu lenguaje elegido.
7. Sube tu avance final al repositorio asignado de GitHub Classroom.

## Criterios generales de evaluación
- Claridad de la propuesta y del problema.
- Viabilidad del alcance con recursos básicos.
- Coherencia entre caso de uso, estructura y pruebas.
- Orden del repositorio y calidad de documentación.
- Calidad del prototipo mínimo (si se implementa).

## Nota importante
**Primero documenta y justifica la idea; después, de forma opcional, implementa un prototipo pequeño.**
