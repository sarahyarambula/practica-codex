# Checklist Breve de Validación

## 1) Checklist de pruebas mínimas
- [ ] Probé al menos 3 casos funcionales.
- [ ] Probé al menos 1 caso con error de entrada.
- [ ] Registré resultados esperados y obtenidos.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Completé `docs/plan_de_pruebas.md`.
- [ ] Verifiqué coherencia entre alcance, caso de uso y pruebas.

## 3) Checklist de ejecución
- [ ] Ajusté `scripts/run.sh` a mi lenguaje.
- [ ] Ejecuté el proyecto desde terminal.
- [ ] Confirmé que no depende de servicios externos.

## 4) Checklist de entrega final
- [ ] Estructura del repositorio limpia y completa.
- [ ] Sin archivos innecesarios ni temporales.
- [ ] Commits con mensajes claros.
- [ ] Entrega publicada en GitHub Classroom.
